## 3. Decisiones tomadas

| Decisión | Alternativa descartada | Razón |
|---|---|---|
| Cache canónico de `Product[]`/`Categoria[]` y mapear a `PosProduct`/`PosCategory` en consumo | Cachear cada vista por separado | Fuente única de verdad evita drift cuando POS y Catálogo ven el mismo dato. El mapeo es trivial. |
| Effect en el store para invalidar al cambiar user | Llamar `store.invalidate()` desde `signOut` | El effect cubre todos los caminos de cambio de user (login, logout, switch), no solo `signOut`. |
| Productos page conserva referencia al `repo` para `deactivateProduct` | Mover deactivate al store | Las mutaciones DB siguen viviendo en el repo (separación clara). El store solo refleja estado local. |
| `withPreloading(PreloadAllModules)` | `QuicklinkPreloader` o sin preloading | Tienda con WiFi estable; ~50 kB extra después del primer paint pinta navegaciones instantáneas. Quicklink requiere lib externa. |
| Reportes: filtrar sesiones por rango en DB (no traer 50 + filter JS) | Conservar `listSessions(50)` | Trabajo desperdiciado y se vuelve peor con más historial. El nuevo método tiene 1 RTT del mismo costo. |
| **NO** migrar agregaciones de Reportes a RPC SQL | Crear `get_daily_report()` en Postgres | Premature optimization para 50–100 ventas/día. Aplicar sólo cuando se mida lentitud real con datos en producción. |
| **NO** aplicar Fase 5 (PWA) | `ng add @angular/pwa` ahora | Plan original explícitamente "POST-MVP, no priorizar hasta tener datos reales". CLAUDE.md §3.4 prohíbe scope creep. SW introduce cache invalidation que puede enmascarar bugs en dev. |
| Mover el store a `features/products/` | Dejarlo en `core/data/` | Auditoría detectó dependencia core → features (violación de dirección). En este codebase la regla es feature → core. |
| Extraer `TtlCache` puro a `src/shared/cache/` | Mantener cache+coalescing duplicado entre `ensureProducts` y `ensureCategorias` | Permite tests unitarios sin setup de Angular y queda reutilizable para futuros caches (p. ej. reporte diario). |
| Extraer `getErrorMessage(error, fallback)` a `src/shared/lib/` | Mantener el ternario inline `error instanceof Error ? error.message : '...'` | 22 ocurrencias en 16 archivos. Centralizar permite manejar `string` y subclases de `Error` correctamente, y deja el sitio listo para logging futuro. |
